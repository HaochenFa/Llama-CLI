# 思维内容处理功能

## 概述

LlamaCLI 现在支持智能处理模型的思维内容。当您使用支持思维功能的模型（如使用 `<think></think>` 标签的模型）时，系统会自动识别并优化显示思考过程。

## 功能特性

### 🧠 自动思维识别
- 自动检测 `<think></think>` 标签
- 实时处理流式输出中的思维内容
- 支持嵌套和多个思维块

### 💭 优化显示体验
- **默认折叠**: 思维内容默认不显示，保持对话流畅
- **智能提示**: 显示简洁的思考状态提示
- **按需查看**: 用户可以主动查看思考过程

### 🎛️ 灵活控制
- 完全的用户控制权
- 可开启/关闭思维显示
- 支持查看历史思维记录

## 使用方法

### 基本使用

当模型开始思考时，您会看到：
```
💭 [思考] 模型正在分析问题... (输入 /think 查看详情)
```

当思考完成时：
```
✨ [思考完成] 输入 /think 查看思考过程
```

### 思维管理命令

在交互式聊天中，您可以使用以下命令：

#### `/think` - 查看最新思维
显示最近一次的思考内容

#### `/think list` - 列出所有思维记录
```
🧠 思维历史:

  1. [14:30:25] 折叠 - 让我思考一下这个问题...
  2. [14:31:02] 折叠 - 等等，让我再仔细考虑一下...

💡 使用 /think <序号> 查看具体内容
```

#### `/think <数字>` - 查看指定思维
例如：`/think 1` 查看第一个思维记录

#### `/think on/off` - 开启/关闭思维显示
- `/think on`: 开启思维内容显示
- `/think off`: 关闭思维内容显示

#### `/think clear` - 清空思维历史
删除所有保存的思维记录

## 技术实现

### 核心组件

1. **ThinkingRenderer** - 思维内容渲染器
   - 管理思维历史
   - 提供格式化显示
   - 支持折叠/展开控制

2. **StreamProcessor** - 流式内容处理器
   - 实时解析流式输出
   - 识别思维标签
   - 分离思维内容和普通内容

### 处理流程

```
流式输入 → StreamProcessor → 识别<think>标签 → 分离内容
                                    ↓
普通内容 → 直接输出到终端    思维内容 → ThinkingRenderer → 存储&管理
```

## 配置选项

### 默认设置
- 思维显示：开启
- 默认状态：折叠
- 历史记录：保留在会话期间

### 自定义配置
您可以通过以下方式自定义行为：
- 使用 `/think on/off` 控制显示
- 使用 `/think clear` 管理历史

## 兼容性

### 支持的模型
- 任何使用 `<think></think>` 标签的模型
- 支持多行思维内容
- 支持嵌套思维标签

### 适配器支持

- ✅ Gemini 适配器
- ✅ Ollama 适配器
- ✅ vLLM 适配器
- ✅ OpenAI 兼容适配器

## 最佳实践

1. **保持默认设置**: 大多数情况下，默认的折叠显示提供最佳体验
2. **按需查看**: 只在需要了解模型思考过程时使用 `/think` 命令
3. **定期清理**: 使用 `/think clear` 清理不需要的思维历史
4. **调试时开启**: 在调试或学习时可以使用 `/think on` 查看实时思维

## 故障排除

### 思维内容没有被识别
- 确认模型输出使用了正确的 `<think></think>` 标签
- 检查标签是否正确闭合

### 思维显示异常
- 使用 `/think off` 然后 `/think on` 重置状态
- 使用 `/think clear` 清空历史记录

### 性能问题
- 定期使用 `/think clear` 清理历史
- 考虑关闭思维显示 (`/think off`) 以提高性能
